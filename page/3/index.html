hadoop 2.7.2集群搭建（7台）

hadoop 2.x简介

hadoop 2.x已经发布了稳定版本了，增加了很多特性，比如HDFS HA、YARN等。

注意：apache提供的hadoop-2.7.2的安装包是在32位操作系统编译的，因为hadoop依赖一些C++的本地库，
所以如果在64位的操作上安装hadoop-2.7.2就需要重新在64操作系统上重新编译

前期准备

集群规划：

主机名 IP 安装的软件 运行的进程
yun01 192.168.1.201 jdk、hadoop NameNode、DFSZKFailoverController
yun02 192.168.1.202 jdk、hadoop NameNode、DFSZKFailoverController
yun03 192.168.1.203 jdk、hadoop ResourceManager
yun04 192.168.1.204 jdk、hadoop ResourceManager
yun05 192.168.1.205 jdk、hadoop、zookeeper DataNode、NodeManager、JournalNode、QuorumPeerMain
yun06 192.168.1.206 jdk、hadoop、zookeeper DataNode、NodeManager、JournalNode、QuorumPeerMain
yun07 192.168.1.207 jdk、hadoop、zookeeper DataNode、NodeManager、JournalNode、QuorumPeerMain

说明：

在hadoop 2.x中通常由两个NameNode组成，一个处于active状态，另一个处于standby状态。Active NameNode对外提供服务，而Standby NameNode则不对外提供服务，仅同步active namenode的状态，以便能够在它失败时快速进行切换。
hadoop 2.x官方提供了两种HDFS HA的解决方案，一种是NFS，另一种是QJM。这里我们使用简单的QJM。在该方案中，主备NameNode之间通过一组JournalNode同步元数据信息，一条数据只要成功写入多数JournalNode即认为写入成功。通常配置奇数个JournalNode
这里还配置了一个zookeeper集群，用于ZKFC（DFSZKFailoverController）故障转移，当Active NameNode挂掉了，会自动切换Standby NameNode为standby状态
​

安装步骤：

安装配置zooekeeper集群（在yun05上操作）

解压： tar -zxvf zookeeper-3.4.8.tar.gz -C /hadoop/

修改配置

vim zoo.cfg 修改：

dataDir=/home/hadoop/cloud/zookeeper-3.4.8/tmp

在最后添加：

保存退出
然后创建一个tmp文件夹

mkdir /home/hadoop/cloud/zookeeper-3.4.8/tmp

再创建一个空文件
touch /home/hadoop/cloud/zookeeper-3.4.8/tmp/myid

最后向该文件写入ID
echo 1 > /home/hadoop/cloud/zookeeper-3.4.8/tmp/myid

将配置好的zookeeper拷贝到其他节点(首先分别在yun05、yun06 的hadoop用户目录下创建一个cloud目录：mkdir /home/hadoop/cloud)

注意：修改yun05、yun06对应/home/hadoop/cloud/zookeeper-3.4.8/tmp/myid内容

yun05： echo 2 > /home/hadoop/cloud/zookeeper-3.4.8/tmp/myid
yun06： echo 3 > /home/hadoop/cloud/zookeeper-3.4.8/tmp/myid

安装配置hadoop集群

解压

tar -zxvf hadoop-2.7.2.tar.gz -C /home/hadoop/cloud/

配置HDFS（hadoop2.x所有的配置文件都在$HADOOP_HOME/etc/hadoop目录下）

将hadoop添加到环境变量中
vim /etc/profile

hadoop 2.x的配置文件全部在$HADOOP_HOME/etc/hadoop下
cd /home/hadoop/cloud/hadoop-2.7.2/etc/hadoop
​

修改hadoop-env.sh

vi hadoop-env.sh

export JAVA_HOME=/home/hadoop/cloud/jdk1.8

修改core-site.xml

vi core-site.xml

修改hdfs-site.xml

vi hdfs-site.xml

修改mapred-site.xml

vi mapred-site.xml

修改yarn-site.xml

vi yarn-site.xml

修改slaves(slaves是指定子节点的位置，因为要在yun01上启动HDFS、在yun03启动yarn，所以yun01上的slaves文件指定的是datanode的位置，yun03上的slaves文件指定的是nodemanager的位置)

配置免密码登陆

首先要配置yun01到yun02、yun03、yun04、yun05、yun06、yun07的免密码登陆

在yun01上生产一对钥匙 ssh-keygen -t rsa 将公钥拷贝到其他节点，包括自己

配置yun03到yun04、yun05、yun06、yun07的免密码登陆

在yun03上生产一对钥匙 ssh-keygen -t rsa 将公钥拷贝到其他节点

注意：两个namenode之间要配置ssh免密码登陆，别忘了配置yun02到yun01的免登陆,在yun02上生产一对钥匙 ssh-keygen -t rsa

将配置好的hadoop拷贝到其他节点

##注意：严格按照下面的步骤

启动zookeeper集群（分别在yun05、yun06、yun07上启动zk）

cd /home/hadoop/cloud/zookeeper-3.4.8/bin/

查看状态：一个leader，两个follower

启动journalnode

方法一：（在yun01上启动所有journalnode，注意：是调用的hadoop-daemons.sh这个脚本，注意是复数s的那个脚本）

cd /home/hadoop/cloud/hadoop-2.7.2

运行jps命令检验，yun04、yun05、yun06上多了JournalNode进程

方法二：（分别在在yun05、yun06、yun07上执行）

cd /home/hadoop/cloud/hadoop-2.7.2

格式化HDFS

在yun01上执行命令: hdfs namenode -format
格式化后会在根据core-site.xml中的hadoop.tmp.dir配置生成个文件，这里我配置的是/hadoop/hadoop-2.7.2/tmp，然后将/hadoop/hadoop-2.7.2/tmp拷贝到yun02的/hadoop/hadoop-2.7.2/下。
scp -r tmp/ yun02:/home/hadoop/cloud/hadoop-2.7.2/

也可以这样，建议 hdfs namenode -bootstrapStandby

格式化ZK(在yun01上执行即可)

hdfs zkfc -formatZK
​

启动HDFS(在yun01上执行)

sbin/start-dfs.sh

启动YARN

(注意：是在yun03上执行start-yarn.sh，把namenode和resourcemanager分开是因为性能问题，因为他们都要占用大量资源，所以把他们分开了，他们分开了就要分别在不同的机器上启动)
sbin/start-yarn.sh
​
hadoop2.7.2配置完毕!!!

http://192.168.1.202:50070 NameNode ‘yun02:9000’ (standby)
